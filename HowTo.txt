#Information
1)The SN Demons start with ports 33101 going up consecutively.
A special script is written to skip the standard ports, which are 33123, 33124 and 33125.
This script, "NodeConfig.sh" functions as entrypoint for the Docker Container
This way, a standard demon can also be ran on the system where the multi container cluster is deployed,
which can be used to acces a wallet, through the cli-wallet.
2)The SN Demons all run with the following flags
--confirm-external-bind
--service-node
--non-interactive
3)A remote Demon is Setup up on a 32core System with 64gb RAM, which is piggy backed by the SN Demons through the following flag
--add-peer 88.218.224.23
4)Lastly, the SN Demons also piggy back, as a sync point to a pool set up by the discord user huser#7983
--add-peer 51.68.191.40

Both of these serveres are in Germany.

Before Creating the Containers with Docker, (which will actually start them after creation)

If such a folder isnt available, the logically course of action is to run at first a single Container, wait it to sync, and then copy the "lmdb" file to the other data folders.

#How the containers work, MUST READ.
The XTNC_SN Containers have two parts.
The 1st part, containing the XTNC Demon, and other needed files, are inside the running container, and cannot be accesed from outside of the system
The 2nd part, the storage of the XTNC Blockchaindata, is accesable from the outside, as it is a folder mounted to the container with the "-v" tag.

When the containers are created, a variable CN is passed, which determines the number of the XTNC SN Container in its creation, which further determines the ports the SN Demon will run with.
The "NodeConfig.sh" script takes this variable, and determines the working ports
Also, when the containers are created, the ports of each of the container must be specified here as well, note that these are exactly the ports the "NodeConfig.sh" determines through the CN variable.

From the list below, one needs to replace only the path 
/home/user/SSD/BlockchainData/XTNC_MD_Data/xtnc01
with one's own datapath. The ports are allready written.
Notice how on container Number 8, the ports used are 33122, then 33126, thus skipping 33123, 33124, and 33125, which are the standard ports.
If a higher than 50 container number is required, one simply has to continue the port numbering consecutively.

sudo docker run -d -e CN=1 -p 33101:33101 -p 33102:33102 -p 33103:33103 -v /home/user/SSD/BlockchainData/XTNC_MD_Data/xtnc01:/XtendCash/SN_Data --name xtnc01 xtnc_md
